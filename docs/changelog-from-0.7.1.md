# Изменения после снимка `v0.7.1`

Сравнение сделано между Repomix-снимком `e025a2fbeec3ba9a474af573c6a28013c3fadeef` от 6 марта 2026 года и текущим состоянием ветки `main` на 5 апреля 2026 года. Сам снимок идет сразу после тега `v0.7.1`, поэтому ниже собраны изменения, появившиеся уже после этой точки.

Это не полный список всех коммитов, а нормализованная сводка по крупным изменениям в продукте, API и инфраструктуре.

## Коротко

- Из репозитория выросла полноценная web-платформа: появились публичный фронтенд, авторизация, web-расписание, админская статистика и отдельная `Document Studio`.
- Telegram-бот получил объединенный поиск `/search` и систему сохраненных пресетов `/search_presets`.
- Блок расписания сильно расширился: оффлайн-фолбэк, фильтры по модулям, WebCal/ICS-подписка, история поиска и синхронизация настроек.
- Проект стал заметно зрелее по эксплуатационной части: новые миграции, health-check эндпоинты, proxy-aware сеть, CI/CD, авто-линтинг и новые тесты.

## Основные изменения

### 1. Web-платформа

- Добавлен новый каталог `main_site_frontend/` с отдельными страницами:
  - лендинг `/`
  - вход `/login`
  - регистрация `/register`
  - web-расписание `/schedule`
  - студия документов `/studio`
  - админская статистика `/stats`
- Появилась полноценная JWT-авторизация для сайта:
  - вход по логину и паролю
  - регистрация web-аккаунтов
  - вход через Telegram Login Widget
  - роли `user` и `admin`
  - получение и обновление пользовательских web-настроек через API
- Добавлена синхронизация предпочтений сайта:
  - сохранение выбранного расписания
  - сохранение фильтров модулей
  - сохранение режима коротких названий
  - при наличии аккаунта настройки теперь могут жить не только в `localStorage`, но и в базе
- Добавлена полноценная web-страница расписания:
  - поиск групп, преподавателей и аудиторий
  - загрузка расписания через новый FastAPI router
  - показ оффлайн-копии, если API вуза недоступно
  - список недавно закэшированных сущностей
  - фильтры по модулям
  - переключение коротких и полных названий дисциплин
  - адаптация под десктоп и мобильный вид
- Добавлена `Document Studio`:
  - quick mode для единичного фрагмента
  - project mode для полноценных проектов
  - поддержка `LaTeX`, `Markdown`, `Mermaid`
  - шаблоны проектов
  - мультифайловые проекты и хранение файлов в БД
  - загрузка бинарных ассетов
  - live preview
  - сборка PDF через worker/Celery
  - экспорт проекта в ZIP
  - отправка собранного PDF себе в Telegram
- Сильно обновлен web-dashboard статистики:
  - админский доступ через JWT вместо старого basic auth
  - более богатый UI с фильтрами, колонками, плотностью таблиц и диапазонами дат
  - диагностическая панель
  - улучшенная leaderboard/activity выдача
  - подготовлена визуальная baseline-проверка интерфейса

### 2. Telegram-бот

- Добавлена новая команда `/search`:
  - объединенный поиск по `matplobblib`
  - одновременный поиск по подключенным GitHub-конспектам
  - включение и отключение источников
  - выбор конкретных репозиториев для GitHub-поиска
  - пагинация и открытие результата в правильном viewer
- Добавлена новая команда `/search_presets`:
  - сохранение поисковых сценариев
  - повторный запуск сохраненного поиска
  - удаление пресетов
  - поддержка нескольких типов пресетов: library, github, schedule, global
- Новые команды встроены в:
  - главное reply-меню
  - bot command list
  - `/help`
- Под unified search и presets уже добавлена отдельная документация в `docs/wiki.md`.
- Существенно переработан расписательный UX в боте:
  - история последних поисков
  - очистка истории
  - фолбэк на локальный кэш, если API вуза недоступно
  - предупреждение пользователя о работе в оффлайн-режиме
  - улучшенный `myschedule`
  - недельные и дневные представления через календарный UI
  - фильтры по типам занятий и по источникам подписки
- Добавлена настройка модулей для подписок на расписание:
  - можно явно отметить, какие модули или подгруппы посещает пользователь
  - расписание и уведомления фильтруются с учетом выбора
  - появилась генерация деталей по составу модулей и преподавателей
- Добавлена WebCal/ICS-подписка на агрегированное расписание:
  - персональный секретный токен календаря
  - ссылка для iOS/Google Calendar
  - возможность перевыпустить секрет
- Расширены админские сценарии:
  - ежедневная сводка по боту
  - настройка времени admin summary
  - обработка pending-предложений по сокращению названий прямо в ежедневном цикле

### 3. Backend, данные и API

- Добавлены новые таблицы и миграции для:
  - `web_accounts`
  - `projects`
  - `project_files`
  - `build_cache`
  - связи web-аккаунта с Telegram-пользователем
  - web preferences
  - `calendar_secret` для ICS-подписки
- В FastAPI появились новые router'ы:
  - `auth_router`
  - `schedule_router`
  - `studio_router`
  - расширенный `stats_router`
  - `calendar_router` для публичного WebCal/ICS
- Обновлена shared-логика расписания:
  - кэш-фолбэк при недоступности внешнего API
  - поиск по локально сохраненным сущностям
  - гибридное определение доступных модулей
  - агрегация расписания для календаря и web-view
- Добавлены и усилены health-check механизмы:
  - `/api/stats/health`
  - отдельный health server в `scheduler_app` на `:9584/health`
- Сеть и внешние интеграции стали устойчивее:
  - поддержка proxy через `PROXY_URL`
  - переход на `socks5h` для корректного DNS-resolve
  - увеличены таймауты для bot/FastAPI/scheduler
  - добавлена работа с `aiohttp_socks`

### 4. Инфраструктура, сборка и качество

- Пересобран pipeline зависимостей:
  - добавлен `requirements.in`
  - обновлен `requirements.txt`
  - обновлены `setup.py` и `pyproject.toml`
- Обновлены Docker- и deploy-артефакты:
  - новые base images
  - обновления `docker-compose.yml` и `docker-compose.prod.yml`
  - новые конфиги для static frontend и reverse proxy
  - добавлен `Caddyfile`
  - добавлен блок `proxy/` с cleaner/config
- Усилен CI/CD:
  - переработан workflow `ci-cd.yml`
  - добавлен `autolint-autofix.yml`
  - добавлен `stats-visual-regression.yml`
  - обновлен `Jenkinsfile.groovy`
- Добавлены новые тесты:
  - `test_auth_flow.py`
  - `test_authorization_guards.py`
  - `test_search_center_helpers.py`
  - `test_stats_leaderboard_api.py`
  - `test_version_bumper.py`
  - визуальный baseline capture для страницы статистики

## Важные практические итоги

- После `v0.7.1` проект стал не только Telegram-ботом, но и самостоятельным web-сервисом с аккаунтами, расписанием и студией документов.
- Поиск в боте больше не разбит на изолированные сценарии: теперь есть единая точка входа `/search` и повторное использование через пресеты.
- Расписание стало заметно устойчивее к сбоям внешнего API за счет кэша, offline-mode и календарной подписки.
- Кодовая база стала гораздо ближе к production-эксплуатации: больше миграций, лучше observability, больше автопроверок и тестового покрытия.
